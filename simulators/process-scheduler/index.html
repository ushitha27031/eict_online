A real OS juggles far more processes than fit in main memory at once, so it keeps the overflow parked in virtual memory on disk. A Ready or Blocked process that gets swapped out is suspended — it still exists, it just isn't resident in RAM — and gets activated back when a slot opens up. Create a few processes, watch main memory fill up on the gauge below, and use Suspend / Activate to move processes across the memory boundary yourself — or let Auto-run do it.
